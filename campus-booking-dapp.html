Setup Guide
Deploy and test on Ganache in 10 minutes
Demo Wallet Setup (4 accounts)
ℹ️ In Ganache, each account starts with 100 ETH. Import these addresses into MetaMask using their private keys.
Contract Configuration
After deploying CampusAssetBooking.sol on Ganache:
1. Copy the contract address
2. Paste it above
3. Connect MetaMask (network: localhost:8545, chainId 1337)
4. Switch accounts to test different roles
Step-by-Step Deployment
1
Install prerequisites
Install Ganache (desktop or CLI: npm i -g ganache), Truffle (npm i -g truffle), and the MetaMask browser extension.
2
Start Ganache
Run ganache --server.port 8545 --chain.chainId 1337 --chain.networkId 1337 --wallet.deterministic. This gives you 10 deterministic accounts with 100 ETH each. Note the first 4 addresses.
3
Deploy the contract
Create a Truffle project, place CampusAssetBooking.sol in contracts/, write a migration, then run:
truffle migrate --network development
Copy the deployed contract address.
4
Configure MetaMask
Add network: RPC http://127.0.0.1:8545, Chain ID 1337. Import the 4 Ganache accounts using their private keys (shown in Ganache UI).
5
Admin setup (Account 0)
Connect with Account 0 (admin). Go to Admin Panel → register Assets → register Accounts 1 and 2 as users. Account 3 remains unregistered to demonstrate rejection.
6
Test all scenarios
Switch to Account 1 → book a slot (pays ETH) → verify on-chain.
Switch to Account 2 → try same slot → expect rejection.
Switch to Account 3 → try any booking → expect "not registered" revert.
Test cancellation with both refund tiers.
